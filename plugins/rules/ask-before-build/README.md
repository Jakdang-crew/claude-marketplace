# ask-before-build

Prevent speculative implementation. When uncertain about design decisions, ask the user first instead of guessing.

## Rules

- Do not create files, scripts, or configurations based on assumptions.
- Confirm with the user before implementing when the direction is unclear.
- Only implement what was explicitly requested — no unrequested extras.

## Scope

Recommended as a user-level rule (`~/.claude/rules/`) to apply across all projects.
